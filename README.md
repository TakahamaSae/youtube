# youtube

## テーマ特化型切り抜き動画の自動生成手法（解析→抽出→合成）

以下の3段構成で実装すると、要件を満たしやすいです。

### 0. 前提（権利・運用）
- 対象チャンネル（中町兄妹 / むくえなちっく / コノリリ）は、**転載・切り抜き条件を明文化している動画のみ**を入力対象にする。
- 動画ごとに `source_url`, `channel`, `license_note`, `download_date` をメタデータとして保存する。

---

### 1. 解析（何を話しているか理解する）
1. YouTube Data APIで動画一覧を取得し、yt-dlpで音声を抽出（`wav` 16kHz mono）。
2. Whisper系ASRで文字起こし（セグメント単位で開始/終了時刻付き）。
3. 文字起こし結果を文単位に再分割し、埋め込み（embedding）を生成してベクトルDBに保存。

**保存例（1発話）**
```json
{
  "video_id": "xxxx",
  "start": 123.4,
  "end": 131.8,
  "text": "最近は恋愛より仕事を優先していて…",
  "embedding": "...",
  "speaker": "unknown"
}
```
※話者を使う運用にする場合は、ASR後に話者分離（diarization）を追加して `speaker` を埋める。

---

### 2. 抽出（テーマ別に自動ピックアップ）
「恋愛」「キャリア」などのテーマごとに、次のハイブリッド抽出が精度・再現率のバランスが良いです。

1. **一次抽出（高速）**  
   キーワード + ベクトル類似検索（例: テーマ説明文を埋め込み化して近傍検索）。
2. **二次判定（高精度）**  
   LLMに「この発話はテーマに関係あるか」を関連度スコア `[0.0, 1.0]` で判定させる（例: 0.7以上を採用）。
3. **品質フィルタ**  
   - 最低発話長（例: 3秒以上）  
   - 信頼度しきい値（ASR信頼度、LLM判定スコア）  
   - 重複・類似発話の除外
4. **つなぎ最適化**  
   前後1〜2秒の文脈を付与して不自然な切れ目を防ぐ。

---

### 3. 合成（新しい動画として書き出す）
1. 抽出セグメントを時系列またはストーリー順に並べる（導入→本編→まとめ）。
2. MoviePy/ffmpegで自動編集：
   - 元動画映像の該当区間を切り出し
   - 音量正規化（ラウドネスを揃える）
   - 必要ならB-roll/背景映像を重ねる
   - テーマ名・出典テロップを挿入
3. 最終出力（例: `mp4 H.264/AAC`）と同時に、出典一覧（URL/タイムスタンプ）を生成。

---

## 推奨システム構成（最小実装）
- **収集**: YouTube Data API + yt-dlp
- **ASR**: Whisper（PoCは medium 以上推奨、本番は large-v3 など）
- **保存**: SQLite/PostgreSQL + ベクトルDB（pgvector/Chroma）
- **抽出判定**: OpenAI/Gemini等のLLM
- **編集**: ffmpeg + MoviePy
- **オーケストレーション**: Python（バッチ実行）
